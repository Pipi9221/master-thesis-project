# Test87 - l_2830 依赖丢失问题

## 问题

`l_2830` (uint32_t const ****) 需要初始化，但其依赖链中的变量不在切片代码中。

## 依赖链

```
l_2830 (uint32_t const ****)
  ↓ = &l_2831
l_2831 (uint32_t const ***)
  ↓ = &l_2832
l_2832 (uint32_t const **)
  ↓ = &l_2833
l_2833 (uint32_t const *)
  ↓ = &g_1488[0][6]
g_1488 (全局变量数组)
```

## 原始代码

```c
const uint32_t *l_2833 = &g_1488[0][6];
const uint32_t **l_2832 = &l_2833;
const uint32_t ***l_2831 = &l_2832;
const uint32_t ****l_2830 = &l_2831;
```

## 切片代码

```c
uint32_t const ****l_2830;  // ❌ 未初始化
uint32_t const *****l_2829 = & l_2830;  // 使用 l_2830
```

## 问题分析

- `g_1488` 不在切片标准中，因此被移除
- 依赖链 `l_2833` → `l_2832` → `l_2831` → `l_2830` 完全丢失
- `l_2830` 无法正确初始化

## 解决方案

由于依赖链完全丢失，使用 `(void*)0` 初始化以满足编译要求。

**注意**: 这不能完全恢复原始行为，但至少能满足编译要求。
