# va_list Type Assignment Error in Sliced Code

## Issue Summary

Frama-C slicing tool generates code with invalid assignment to array type when slicing programs with variable arguments (`...`). The sliced program attempts to assign a pointer to `va_list` which is an array type, causing compilation errors.

## Source Program

The source program `test.c` contains functions that use variable arguments:

```c
// test.c
#include <stdio.h>
#include <stdarg.h>

static int g_sum = 0;

int sum_with_side_effect(int count, ...) {
    va_list args;
    va_start(args, count);
    int total = 0;
    for (int i = 0; i < count; i++) {
        int val = va_arg(args, int);
        total += val;
        g_sum += val;
    }
    va_end(args);
    return total;
}

int sum_simple(int count, ...) {
    va_list args;
    va_start(args, count);
    int total = 0;
    for (int i = 0; i < count; i++) {
        total += va_arg(args, int);
    }
    va_end(args);
    return total;
}

int main(void) {
    int val = 10;
    
    // Variable argument function calls
    int r1 = sum_with_side_effect(3, 10, 20, 30);
    
    // Multiple calls
    int r2 = sum_simple(4, 1, 2, 3, 4);
    int r3 = sum_with_side_effect(2, val, val + 10);
    
    // In expressions
    int r4 = sum_with_side_effect(2, r1, r2) + sum_simple(2, r3, val);
    
    // In conditionals
    if (sum_with_side_effect(1, val) > 50) {
        val += 10;
    }
    
    int result = val + r1 + r2 + r3 + r4 + g_sum;
    printf("Result: %d\n", result);
    return result;
}
```

This program compiles and runs successfully:
```bash
$ gcc -o test_original test.c && ./test_original
Result: 390
$ echo $?
134
```

## Slice Command

The slicing command used:
```bash
frama-c test.c \
  -slice-value result \
  -then-on 'Slicing export' \
  -print \
  -ocode test_sliced.c
```

## Sliced Program

The sliced program generated by Frama-C (`test_sliced.c`):

```c
/* Generated by Frama-C */
#include "stdarg.h"
#include "stddef.h"
#include "stdio.h"
static int g_sum = 0;
int sum_with_side_effect_slice_1(int count, void * const *__va_params)
{
  va_list args;
  args = __va_params;  // ❌ Error: cannot assign to array type
  int total = 0;
  {
    int i = 0;
    while (i < count) {
      {
        int tmp;
        tmp = *((int *)*args);  // ❌ Error: cannot convert to pointer type
        args ++;                // ❌ Error: lvalue required as increment operand
        int val = tmp;
        total += val;
        g_sum += val;
      }
      i ++;
    }
  }
  return total;
}

// Similar errors in sum_simple_slice_1...

int main(void)
{
  // ... main function with transformed variable arguments
}
```

## What Happens When Running

1. **Compilation fails** with multiple errors:
   ```bash
   $ gcc -o test_sliced test_sliced.c
   test_sliced.c:9:8: error: assignment to expression with array type
   test_sliced.c:16:9: error: cannot convert to a pointer type
   test_sliced.c:17:14: error: lvalue required as increment operand
   ```

2. **Root cause**: In most C implementations, `va_list` is an array type:
   ```c
   typedef char va_list[24];  // Example on x86_64 Linux
   ```

3. **Problem**: Frama-C converts `...` parameters to `void * const *__va_params` (pointer type) but tries to assign it to `va_list args` (array type), which violates C language rules.

4. **Additional issues**: Even if assignment worked, the code attempts to perform pointer arithmetic (`args++`) on an array type, which is also invalid.

## Root Cause

The slicing tool:
- ✅ Converts `...` parameters to `void * const *__va_params` (pointer type)
- ❌ Attempts to assign pointer to `args` which is `va_list` (array type)
- ❌ Violates C language syntax rules for array assignment
- ❌ Attempts invalid pointer arithmetic on array type

Arrays cannot be assigned directly in C:
```c
int arr1[10];
int arr2[10];
arr1 = arr2;  // ❌ Error: arrays cannot be assigned
```

## Workaround

### Option 1: Use memcpy (Partially works)
```c
#include <string.h>
int sum_with_side_effect_slice_1(int count, void * const *__va_params) {
    va_list args;
    memcpy(&args, &__va_params, sizeof(va_list));  // ✅ Use memcpy for assignment
    // But pointer arithmetic still problematic
}
```

### Option 2: Preserve original va_start/va_end (Recommended)
```c
int sum_with_side_effect_slice_1(int count, ...) {
    va_list args;
    va_start(args, count);  // ✅ Preserve original approach
    // ... rest of implementation using va_arg, va_end
    va_end(args);
}
```
