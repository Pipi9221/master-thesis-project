# Test19 - l_2359 依赖丢失问题

## 问题

`l_2359[5][9][5]` 是一个 `const` 数组，需要初始化，但其依赖的全局变量 `g_832` 和 `g_2360` 不在切片代码中。

## 原始代码

```c
const int32_t *l_2359[5][9][5] = {
    // 225个元素，每个都是 &g_832, &g_2360, 或 (void*)0
    // ... 复杂的初始化
};
l_2359[3][0][2] = func_41(...);  // 后来被赋值
```

## 切片代码

```c
int32_t const *l_2359[5][9][5];  // ❌ 未初始化，但依赖的 g_832 和 g_2360 不在切片中
```

## 解决方案

由于 `l_2359[3][0][2]` 后来被赋值，整个数组的初始值实际上不重要。为了满足 C 语言的 `const` 数组初始化要求，可以用 `(void*)0` 初始化所有元素。

**注意**: 这不能完全恢复原始行为，因为 `l_2359` 的其他元素可能在某些控制流中被使用（虽然不在切片中）。
