# Issue Submission Template

**Title**: Slicing loses initialization of const local arrays used in computations

**Type**: Bug Report (Miscompilation)

**Component**: Slicing

---

## Description

When slicing on a variable that depends on a `const` local array, Frama-C correctly preserves the array declaration but removes its initialization. This causes the sliced program to read uninitialized memory, producing incorrect results.

## Steps to Reproduce

1. Create a source file with a `const` local array initialized and used in calculations
2. Run: `frama-c source.c -slice-value TARGET_VAR -then-on 'Slicing export' -print -ocode sliced.c`
3. Observe: Array initialization is missing in `sliced.c`
4. Compile and run both programs: original produces correct value, sliced produces incorrect value

## Expected Behavior

The sliced program should preserve array initialization since array elements are used in the computation.

## Actual Behavior

Array initialization is removed, causing uninitialized memory reads.

## Impact

- **Severity**: High - sliced program produces incorrect results
- **Correctness**: Violates program slicing semantics
- **Undefined Behavior**: Reading uninitialized memory
